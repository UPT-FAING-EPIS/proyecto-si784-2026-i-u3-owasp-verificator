# Repositorios Públicos en GitHub para Probar el Escáner OWASP

Puedes usar las siguientes URLs de repositorios de GitHub en el formulario del panel web para evaluar cómo funciona el escaneo completo de proyectos públicos:

## 1. Repositorios de Prueba con Vulnerabilidades (Fines Educativos)

* **OWASP NodeGoat (Node.js)**
  - URL: `https://github.com/OWASP/NodeGoat`
  - *Descripción:* Aplicación vulnerable oficial de OWASP para aprender seguridad en Node.js.

* **Damn Vulnerable Web Application - DVWA (PHP)**
  - URL: `https://github.com/digininja/DVWA`
  - *Descripción:* Un clásico entorno de prueba vulnerable construido en PHP.

* **Django Defectuous App (Python)**
  - URL: `https://github.com/saurabhbd/Vulnerable-Django-App`
  - *Descripción:* Una pequeña aplicación Django de prueba configurada intencionalmente con fallos.

## 2. Repositorios de Código Limpio / Producción

* **Este Mismo Proyecto (Python/FastAPI)**
  - URL: `https://github.com/UPT-FAING-EPIS/proyecto-si784-2026-i-u1-verificador-de-cumplimiento-de-owasp`
  - *Descripción:* Úsalo para probar cómo el escáner se evalúa a sí mismo.

* **FastAPI Realworld App (Python)**
  - URL: `https://github.com/nsidnev/fastapi-realworld-example-app`
  - *Descripción:* Un backend limpio en FastAPI siguiendo buenas prácticas.
