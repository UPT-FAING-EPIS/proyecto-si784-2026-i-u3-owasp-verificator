Verificador de Cumplimiento OWASP

Plataforma profesional de auditoría de seguridad con análisis estático, escaneo de URLs, repositorios GitHub y gestión SOC.

Analizar Ahora

Objetivo del Sistema

El Verificador de Cumplimiento OWASP es una plataforma profesional de auditoría de seguridad diseñada para equipos SOC, analistas, desarrolladores y administradores. Detecta vulnerabilidades en código fuente, endpoints web y repositorios GitHub contra los patrones del OWASP Top 10, entregando hallazgos accionables, puntuación de riesgo y guías de mitigación. Soporta gestión multi-usuario con roles diferenciados (Usuario / Administrador SOC) y se integra con herramientas externas mediante API REST.

Modos de Análisis

Código Fuente

Pega código directamente en el editor. Compatible con Python, JavaScript, Java, PHP, Go, Ruby, C/C++, TypeScript, Kotlin, Scala, SQL, Bash, Terraform, YAML y Dockerfile.

URL / Web App

Analiza servidores y aplicaciones web activas. Verifica cabeceras HTTP de seguridad, configuración TLS, exposición de información de servidor y configuraciones inseguras.

Archivo

Sube un archivo de código directamente desde tu equipo. Soporta los mismos formatos que el análisis de código. Análisis estático inmediato sin exposición del código en la red.

Repositorio GitHub

Escanea repositorios públicos o privados de GitHub por URL o owner/repo. Los usuarios con cuenta pueden guardar su token PAT para no ingresarlo cada vez.

Características del Sistema

Detección de Vulnerabilidades OWASP Top 10

  • OWASP-A01: Control de Acceso Roto — Detección de rutas sin autenticación y privilegios incorrectos.
  • OWASP-A02: Exposición de Secretos — Detecta contraseñas, API keys, tokens y credenciales hardcodeadas.
  • OWASP-A03: Funciones Peligrosas — Identifica eval(), exec(), pickle.loads() y subprocesos inseguros.
  • OWASP-A04: Diseño Inseguro — Verifica manejo inseguro de entrada de usuario sin sanitizar.
  • OWASP-A05: Headers Faltantes — Comprueba cabeceras HTTP de seguridad críticas (CSP, HSTS, X-Frame-Options, etc.).
  • OWASP-A06: Componentes Vulnerables — Detección de dependencias y versiones con vulnerabilidades conocidas.
  • OWASP-A09: Logging Insuficiente — Ausencia de manejo de errores y registro de eventos críticos.

Sistema de Roles (Usuario / Admin SOC)

  • Usuario: Puede analizar código, URLs, archivos y repos. Tiene su dashboard personal con historial, estadísticas, gráficos de tendencias, distribución de severidades y gestión de su token GitHub.
  • Administrador SOC: Visualiza el panel global de seguridad corporativo con todos los análisis, Grado de Cumplimiento, Matriz de Severidad y Logs de Actividad. Gestiona cuentas (crear admins, eliminar usuarios) desde la pestaña Cuentas SOC.
  • Invitado (sin cuenta): Puede analizar repos públicos y código directamente, con aviso de registro para guardar token y conservar historial.

Dashboard y Reportes

  • Panel Usuario: Estadísticas personales (total análisis, score promedio, hallazgos críticos, totales). Gráfico de tendencia de seguridad y doughnut de distribución de severidades.
  • Panel Admin (SOC): Grado de Cumplimiento Corporativo (A–F), Matriz de Severidad vs. Frecuencia global, Historial de Reportes de todos los usuarios, Logs de Actividad Reciente.
  • Reportes Individuales: Vista detallada por análisis: hallazgos con evidencia, severidad, penalización de score y remediación específica. Exportación PDF y JSON.
  • Historial de Escaneos: Cada usuario accede a su propio historial; el admin ve todos los análisis de la plataforma.

Integraciones y API

  • API REST: Endpoints documentados con Swagger/OpenAPI (/docs) para integración en pipelines CI/CD.
  • Token de GitHub (PAT): Guardado seguro en cuenta de usuario para escanear repositorios privados sin ingresar el token cada vez.
  • Tokens de API: Generación de tokens de integración propios para uso programático de la API desde herramientas externas.
  • Extensión de Chrome: Extensión oficial v1.0.3 para analizar desde el navegador directamente, con acceso rápido al verificador.
  • Wiki OWASP: Referencia completa integrada de las 10 categorías de vulnerabilidades con ejemplos y mitigaciones.

Sistema de Puntuación

El sistema calcula una puntuación de 0 a 100 basada en la severidad acumulativa de los hallazgos detectados. El score representa el nivel de cumplimiento de seguridad del objetivo analizado:

90–100

Grado A — Excelente

80–89

Grado B — Bueno

70–79

Grado C — Aceptable

60–69

Grado D — Riesgo

<60

Grado F — Crítico

  • Vulnerabilidad HIGH: −30 puntos por hallazgo
  • Vulnerabilidad MEDIUM: −15 puntos por hallazgo
  • Vulnerabilidad LOW: −5 puntos por hallazgo

Casos de Uso

Desarrolladores

Verificar código antes de commits para identificar y remediar vulnerabilidades OWASP tempranamente en el ciclo de desarrollo.

Equipos SOC

Gestión centralizada del cumplimiento de seguridad corporativo. Panel global con métricas, matriz de riesgo y administración de analistas.

Estudiantes

Aprender sobre vulnerabilidades OWASP de forma práctica con retroalimentación inmediata y explicaciones técnicas por hallazgo.

CI/CD Pipeline

Integración vía API REST con pipelines de integración continua (GitHub Actions, Jenkins) para auditoría automática en cada build.

Code Review

Automatizar parte del proceso de revisión de código enfocándose en vulnerabilidades conocidas del OWASP Top 10.

Startups

Verificación de seguridad ágil y económica para aplicaciones nuevas antes del despliegue en producción.
