A01: Control de Acceso Roto RIESGO ALTO

¿Qué significa? Ocurre cuando el sistema no valida si un usuario realmente tiene permiso para acceder a una sección, ruta o dato específico. Quien no tiene privilegios logra evadir las restricciones.

Endpoints de API sin Autenticación OWASP-A01-NO-AUTH

Dejar una función o ruta del servidor expuesta públicamente a Internet por descuido, sin requerir tokens o credenciales de inicio de sesión.

Inseguro (Python)
@app.get("/api/user/private-data")
def get_private_data():
    # CUALQUIER persona en internet puede consultar esto
    return db.fetch_secrets()
Seguro (Python)
@app.get("/api/user/private-data", dependencies=[Depends(auth_verifier)])
def get_private_data(current_user: User = Depends(get_current_user)):
    # Solo usuarios logueados válidos pueden acceder
    return db.fetch_secrets_for(current_user.id)
Comprobación de Rol Hardcodeada OWASP-A01-HARDCODED-ROLE

Comparar strings de roles estáticos directamente en el código de negocio, dificultando la auditoría y exponiendo a fallas por manipulación del cliente.

Inseguro (JavaScript)
if (user.role === 'admin') {
    // Si el usuario edita su perfil local en el navegador,
    // podría cambiar su rol a 'admin' y saltar la validación
    showAdminDashboard();
}
Seguro (JavaScript)
// Consultar la base de datos o verificar firma criptográfica del JWT en el backend
const hasPermission = await checkUserPermission(user.id, 'ACCESS_ADMIN_PANEL');
if (hasPermission) {
    showAdminDashboard();
}
Acceso de Red Totalmente Abierto (CIDR 0.0.0.0/0) OWASP-A01-OPEN-CIDR

Configurar accesos o reglas de firewall abiertos a todo internet (0.0.0.0/0) para servicios que manejan datos sensibles o administrativos.

Inseguro (Terraform)
resource "aws_security_group_rule" "inbound_all" {
  type        = "ingress"
  from_port   = 22
  to_port     = 22
  protocol    = "tcp"
  cidr_blocks = ["0.0.0.0/0"] # Puerto SSH abierto a todo el mundo
}
Seguro (Terraform)
resource "aws_security_group_rule" "inbound_ssh" {
  type        = "ingress"
  from_port   = 22
  to_port     = 22
  protocol    = "tcp"
  cidr_blocks = ["203.0.113.50/32"] # SSH limitado a la IP de la VPN
}

A02: Fallas Criptográficas RIESGO ALTO

¿Qué significa? Proteger de forma deficiente la información confidencial (como claves, contraseñas, tokens o números de tarjeta) al transmitirla o almacenarla, usando algoritmos obsoletos o dejándola en texto plano.

Secretos o Credenciales Quemados (Hardcoded) OWASP-A02-HARDCODED-SECRET

Escribir contraseñas, llaves de API o secretos de cifrado directamente en variables del código fuente que van al repositorio.

Inseguro (JavaScript)
// Si subes esto a GitHub, cualquiera sabrá tu contraseña
const DB_PASSWORD = "SuperPasswordSegura123";
const stripe = require('stripe')('sk_test_51...api_key');
Seguro (JavaScript)
// Cargar credenciales desde variables de entorno locales del servidor
const DB_PASSWORD = process.env.DATABASE_PASSWORD;
const stripe = require('stripe')(process.env.STRIPE_SECRET_KEY);
Algoritmo de Hash Criptográfico Débil OWASP-A02-WEAK-HASH

Uso de hashes MD5 o SHA-1 que son vulnerables a colisiones y pueden ser rotos fácilmente mediante ataques de diccionario rápidos.

Inseguro (Python)
import hashlib
# MD5 está roto, se descifra en milisegundos con tablas arcoíris
hashed_pw = hashlib.md5(password.encode()).hexdigest()
Seguro (Python)
import bcrypt
# Bcrypt es lento por diseño y usa salt, haciéndolo inmune a ataques rápidos
hashed_pw = bcrypt.hashpw(password.encode(), bcrypt.gensalt())

A03: Inyección RIESGO ALTO

¿Qué significa? Permitir que datos proveídos externamente por un usuario sean interpretados directamente como comandos por el intérprete del servidor, base de datos o sistema operativo.

Inyección SQL (SQLi) OWASP-A03-SQLI

Concatenar parámetros directamente a strings de sentencias SQL, permitiendo a un atacante inyectar comandos para robar o destruir la base de datos.

Inseguro (Python)
# Si el usuario escribe: ' OR '1'='1 como username,
# iniciará sesión sin contraseña.
query = f"SELECT * FROM users WHERE user = '{username}' AND pass = '{password}'"
cursor.execute(query)
Seguro (Python)
# Consultas parametrizadas (los datos nunca se interpretan como código SQL)
query = "SELECT * FROM users WHERE user = %s AND pass = %s"
cursor.execute(query, (username, password))
Cross-Site Scripting (XSS) OWASP-A03-XSS

Colocar código proveído por el usuario directamente en el DOM/HTML de la página sin sanitizar, lo que permite a atacantes ejecutar scripts maliciosos en los navegadores de otros usuarios.

Inseguro (JavaScript)
// Si userParam contiene <script>stealCookies()</script>,
// se ejecutará automáticamente.
document.getElementById('welcome').innerHTML = "Hola " + userParam;
Seguro (JavaScript)
// Usar textContent (trata la entrada estrictamente como texto, no HTML)
document.getElementById('welcome').textContent = "Hola " + userParam;
Path Traversal (Salto de Directorio) OWASP-A03-PATH-TRAVERSAL

Utilizar concatenaciones para resolver rutas de lectura o escritura de archivos, permitiendo que un atacante escape a carpetas del sistema usando '../'.

Inseguro (JavaScript)
// Si el usuario solicita: ../../../etc/passwd,
// podrá descargar información sensible del servidor.
const filePath = path.join(__dirname, 'public', req.query.filename);
fs.readFile(filePath, (err, data) => { ... });
Seguro (JavaScript)
// Validar y asegurar que la ruta resuelta esté dentro de la carpeta pública
const safeDir = path.resolve(__dirname, 'public');
const filePath = path.resolve(safeDir, req.query.filename);
if (!filePath.startsWith(safeDir)) {
    return res.status(403).send("Acceso denegado");
}
fs.readFile(filePath, (err, data) => { ... });
SQL Dinámico Inseguro OWASP-A03-DYNAMIC-SQL

Construir sentencias SQL de forma dinámica concatenando variables de entrada directamente dentro de consultas SQL directas o procedimientos almacenados.

Inseguro (SQL)
-- Consulta construida concatenando strings diretamente
DECLARE @query NVARCHAR(MAX);
SET @query = 'SELECT * FROM users WHERE username = ''' + @user_input + '''';
EXEC(@query);
Seguro (SQL)
-- Usar consultas parametrizadas tipadas con sp_executesql
DECLARE @query NVARCHAR(MAX);
SET @query = 'SELECT * FROM users WHERE username = @user';
EXEC sp_executesql @query, N'@user NVARCHAR(50)', @user = @user_input;

A04: Diseño Inseguro RIESGO MEDIO

¿Qué significa? Fallas arquitectónicas cometidas en las fases iniciales de diseño y planeación del software. Por ejemplo, confiar la lógica de seguridad únicamente al cliente o documentar debilidades en los comentarios.

Notas TODO de Seguridad Pendientes OWASP-A04-TODO-SECURITY

Comentarios en el código (`TODO` o `FIXME`) donde se documenta que falta implementar controles de seguridad antes de pasar a producción.

Inseguro (Python)
def check_access(user):
    # TODO: validar permisos aquí, de momento todos son admin
    return True
Seguro (Python)
def check_access(user):
    # Lógica de seguridad implementada y validada en su totalidad
    return user.is_authenticated and user.has_role('admin')

A05: Configuración Incorrecta de Seguridad RIESGO MEDIO

¿Qué significa? Omitir o deshabilitar cabeceras de seguridad, dejar servicios por defecto activos, o configurar las cookies de sesión sin las banderas criptográficas adecuadas.

Modo Debug Habilitado en Producción OWASP-A05-DEBUG-ACTIVE

Habilitar el modo de depuración en servidores de producción, lo que expone trazas de código y variables de entorno ante cualquier fallo.

Inseguro (Python)
# Flask / Django con debug encendido en producción
app.run(debug=True)
Seguro (Python)
# Cargar el modo debug desde configuración
import os
app.run(debug=os.getenv("APP_DEBUG", "False").lower() == "true")
Cookies de Sesión Inseguras OWASP-A05-INSECURE-COOKIE

Crear cookies de sesión sin las propiedades HttpOnly y Secure, posibilitando su robo mediante XSS o intercepción de red.

Inseguro (JavaScript)
// Cookie legible por JS y transmitida por HTTP plano
res.cookie('session_id', '12345', { secure: false });
Seguro (JavaScript)
// Cookie inaccesible a JS y solo transmitida bajo HTTPS cifrado
res.cookie('session_id', '12345', { httpOnly: true, secure: true, sameSite: 'strict' });
Contenedor ejecutado como Root OWASP-A05-DOCKER-ROOT

Ejecutar procesos dentro de contenedores Docker con privilegios de root por defecto, posibilitando ataques de escape de contenedor hacia el host.

Inseguro (Dockerfile)
FROM node:18
# Ejecuta todo el código y la app como usuario root por defecto
COPY . /app
WORKDIR /app
CMD ["node", "index.js"]
Seguro (Dockerfile)
FROM node:18
# Crear un usuario sin privilegios y cambiar a él
RUN groupadd -r app && useradd -r -g app appuser
COPY . /app
RUN chown -R appuser:app /app
USER appuser
WORKDIR /app
CMD ["node", "index.js"]
Escalabilidad de Privilegios en Kubernetes OWASP-A05-K8S-PRIV-ESC

Permitir que los procesos de un contenedor pidan más privilegios de los que ya tienen en el cluster de Kubernetes.

Inseguro (YAML)
securityContext:
  allowPrivilegeEscalation: true # Permite escalar privilegios
Seguro (YAML)
securityContext:
  allowPrivilegeEscalation: false # Bloquea escalada de privilegios

A06: Componentes Vulnerables y Desactualizados RIESGO MEDIO

¿Qué significa? Utilizar bibliotecas externas, frameworks o módulos desactualizados que tengan vulnerabilidades de seguridad documentadas públicamente (CVEs).

Importación de Componentes Obsoletos OWASP-A06-DEP-VULN

Importación de librerías en versiones conocidas por fallas críticas de seguridad.

Inseguro (requirements.txt)
# Versión antigua con fallas de inyección de cabeceras
requests==2.19.0
django==2.2.1
Seguro (requirements.txt)
# Mantener paquetes actualizados
requests>=2.31.0
django>=4.2.8

A07: Fallas en Identificación y Autenticación RIESGO ALTO

¿Qué significa? Implementar mal la autenticación de usuarios. Esto incluye validar credenciales de forma débil, no proteger los intentos de fuerza bruta, o usar números no seguros para generar tokens.

Comparaciones de Contraseñas Vulnerables OWASP-A07-WEAK-PASS-CHECK

Comparar contraseñas mediante operadores simples sin comparar en tiempo constante, posibilitando ataques de canal lateral (timing attacks).

Inseguro (JavaScript)
// Comparación simple (termina antes si los primeros caracteres difieren)
if (inputPassword === savedPassword) { ... }
Seguro (JavaScript)
// Usar criptografía de tiempo constante para evitar timing attacks
const crypto = require('crypto');
if (crypto.timingSafeEqual(Buffer.from(input), Buffer.from(saved))) { ... }
ID de Sesión Predecibles OWASP-A07-WEAK-SESSION

Generar tokens de seguridad o IDs de sesión mediante funciones aleatorias básicas del sistema que son predecibles.

Inseguro (JavaScript)
// Math.random() no es criptográficamente seguro
const sessionId = Math.random().toString(36).substring(2);
Seguro (JavaScript)
// Usar generadores criptográficos seguros (CSPRNG)
const crypto = require('crypto');
const sessionId = crypto.randomBytes(32).toString('hex');

A08: Fallas en Integridad de Software y Datos RIESGO ALTO

¿Qué significa? Confiar en datos, tokens (como JWT) o paquetes descargados de fuentes externas sin verificar previamente su procedencia o si fueron alterados.

Deserialización Insegura OWASP-A08-UNSAFE-DESERIALIZATION

Deserializar datos complejos (objetos Python Pickle o YAML) provistos de fuentes no confiables, lo que permite instanciar código malicioso (RCE).

Inseguro (Python)
import pickle
# Si user_input es manipulado, ejecutará comandos del sistema
user_data = pickle.loads(user_input)
Seguro (Python)
import json
# JSON solo permite tipos de datos planos, no clases ni código dinámico
user_data = json.loads(user_input)
JWT con Algoritmo de Firma 'none' OWASP-A08-UNSIGNED-JWT

Aceptar JSON Web Tokens que especifiquen en su cabecera que no están firmados (`algorithm: "none"`), permitiendo suplantar identidades.

Inseguro (JavaScript)
// Decodificar sin verificar la firma criptográfica
const decoded = jwt.decode(token);
Seguro (JavaScript)
// Forzar algoritmo robusto de firma y validar con clave secreta
const verified = jwt.verify(token, SECRET_KEY, { algorithms: ['HS256'] });

A09: Fallas de Registro y Monitoreo RIESGO BAJO

¿Qué significa? No registrar eventos críticos de seguridad (como inicios de sesión fallidos o errores del sistema), capturar errores silenciosamente (except pass), o registrar información confidencial en los logs públicos.

Captura Silenciosa de Excepciones OWASP-A09-SILENT-EXCEPT

Ocultar errores de código silenciando las excepciones sin dejar ningún rastro o registro del fallo.

Inseguro (Python)
try:
    process_payment()
except Exception:
    # Si falla la transacción, nadie se enterará del error
    pass
Seguro (Python)
import logging
try:
    process_payment()
except Exception as e:
    # Registrar el fallo y la traza para auditoría
    logging.error(f"Fallo en pago: {str(e)}", exc_info=True)
    raise

A10: Server-Side Request Forgery (SSRF) RIESGO ALTO

¿Qué significa? Cuando la aplicación web realiza peticiones de red hacia un servidor externo a petición del usuario, sin validar si la URL de destino apunta a la red interna del propio servidor (como localhost).

Petición HTTP Saliente sin Filtrar OWASP-A10-SSRF-REQUESTS

Hacer peticiones directas a una URL provista en el request, permitiendo a un atacante escanear la red interna corporativa.

Inseguro (Python)
import requests
def fetch_avatar(user_url):
    # Si user_url es: http://127.0.0.1:8500/admin,
    # el servidor consultará su propio panel interno.
    return requests.get(user_url).content
Seguro (Python)
from urllib.parse import urlparse
def fetch_avatar(user_url):
    parsed = urlparse(user_url)
    # Validar que no apunte a IPs locales ni al loopback
    if parsed.hostname in ["localhost", "127.0.0.1", "169.254.169.254"]:
        raise ValueError("URL no permitida")
    return requests.get(user_url).content
